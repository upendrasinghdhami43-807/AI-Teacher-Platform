# AI Teacher Platform — Lesson JSON Schema

## Overview

The `Lesson` object is the core data structure for all AI-generated lessons in the platform. It contains:
- Lesson metadata (title, language, level, mode)
- An array of `LessonFrame` objects that define what appears on the whiteboard at each point in time
- Voice segments for multilingual narration synced to each frame
- An `audio_url` pointing to the stitched audio file
- A `cache_key` for semantic deduplication

## TypeScript Interface

```typescript
interface Lesson {
  lesson_id: string;         // UUID v4
  mode: LessonMode;          // 'ai_teacher' | 'course' | 'smart_pdf'
  title: string;             // Human-readable lesson title
  summary?: string;          // 1-2 sentence summary
  source: LessonSource;      // What triggered this lesson
  language: Language;         // 'en' | 'ne' | 'hi' | 'ne_en'
  level: Level;              // 'basic' | 'medium' | 'advanced'
  version: number;           // Starts at 1, increments on regeneration
  created_at: string;        // ISO 8601
  duration_sec: number;      // Total lesson duration in seconds
  cache_key: string;         // For semantic caching: "topic:lang:level"
  audio_url: string;         // URL to stitched audio file
  frames: LessonFrame[];     // Ordered array of whiteboard frames
}
```

## Frame Structure

Each frame represents a single "board state" — what the whiteboard shows during a specific time window.

```typescript
interface LessonFrame {
  frame_id: number;            // Sequential, starting from 1
  start_sec: number;           // When this frame begins (seconds)
  duration_sec: number;        // How long this frame is shown
  board: BoardAction;          // What to draw/write on the board
  voice_segments: VoiceSegment[]; // What the AI teacher says during this frame
}
```

## Stage 2 Pipeline

In Stage 2, the lesson JSON is generated by a three-step pipeline:

1. **Director Agent** (Gemini) — Receives topic + level + language → outputs a lesson plan with frame outlines
2. **Script Agent** (Gemini) — Receives director plan → outputs detailed board actions + voice scripts per frame
3. **BoardMapper** — Converts script output into the final `Lesson` JSON with proper timing and positioning

Voice segments are sent to **Azure TTS** (or Paaila TTS for Nepali), stitched into a single `audio_url`.

The `cache_key` prevents regenerating identical lessons. Format: `{normalized_topic}:{language}:{level}`

## Notes for Stage 2 Integration

- **Frame timing**: `start_sec` and `duration_sec` must be contiguous — frame N+1 starts at frame N's `start_sec + duration_sec`
- **Voice segment ordering**: within a frame, segments are played sequentially in array order
- **Cache key format**: `{mode}:{normalized-key}:{language}:{level}` — used for semantic deduplication
- **Audio stitching**: individual voice segments are synthesized separately via Azure TTS, then concatenated into the single `audio_url`
- **Board positioning**: `position` coordinates are in a 800×600 virtual canvas space, scaled to actual whiteboard dimensions
